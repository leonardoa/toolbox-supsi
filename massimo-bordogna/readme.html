Mail CO2

Every day millions of people send large quantities of e-mails without realising the climate impact of their actions. In fact, each email sent, depending on the weight and length of the content, consumes a certain amount of CO2 which, when viewed on a global scale, reaches significant numbers. This tool represents an email management platform that makes users aware of the CO2 consumption resulting from sending emails.

The result is obtained by giving some feedback depending on the weight of the email sent, the feedback interact with the computer system making the user experience slower, more unpleasant or annoying in order to raise awareness of the issue of CO2 production via e-mail.

BASE CODE

The interaction between the code and the computer system is reached thanks to a javascrpit library calld RobotJS, to start my project I had a simple base code with 5 interactions.

In particular there are 5 buttons which, if clicked, will trigger a code that, exploiting the external library RobotJS, will interact with the system of the computer giving back different outputs.

The following code for example is the one that describes the movement as a sine wave that the mouse arrow does when the button "mouse" is clicked.

// Move the mouse across the screen as a sine wave.
if (msg == "on") {
// Speed up the mouse.
robot.setMouseDelay(2);

var twoPI = Math.PI * 2.0;
var screenSize = robot.getScreenSize();
var height = screenSize.height / 2 - 10;
var width = screenSize.width;

for (var x = 0; x < width; x++) {
y = height * Math.sin((twoPI * x) / width) + height;
robot.moveMouse(x, y);
}

FIRST VERSION

The first version of the prototypen is the merge between the base code and the figma layout. There are two main layouts: the one to send emails and the one that stores them. When you send an email you'll surely produce some CO2 so the output in this page is always negative. The pages that stores emails (inbox, draft, bin, ...) produces CO2 simply keeping them saved so, when you delete some of them you reduce the pollution so the output is positive.

More precisely in this first version, if you send a light email the output will be an annoying mouse movement for a few seconds, if you send an heavy email (so a lot of pollution) the output will be the mouse movement and the reduction of the screen luminosity (reducing the energy consumed by the computer). If you delete a stored email the screen luminosity will increase, for each email deleted it increases by one step.

FINAL VERSION

The final version presents some little improvements in the layout and 4 different levels of outputs depending on the weight of the email sent:
1 an email with a small text and no file attached will produce an instant reduction of the screen
   luminosity by one step
2 an email with a long text and no file attached will force the mouse in a circolar movement for 15
   seconds, simulating a loading output that forces the user to pause his activity
3 an email with a small text and at least one file attached will produce a gradual reduction of the
   screen luminosity by seven steps
4 an email with a long text and at least one file attached will produce a random change of the
   screen luminosity (generally negative) and a random mouse movement that forces the user to
   pause his activity for a certain amount of time


In this example there is the code that forces the mouse to move in a circle path for 15 seconds when triggered:

else if (msg == "circle") {
robot.setMouseDelay(10);
const screenSize = robot.getScreenSize();
const height = screenSize.height;
const width = screenSize.width;
let elapsedTime = 0;

function moveMouseInCircle() {
const centerX = Math.floor(width / 2); // Center X coordinate
const centerY = Math.floor(height / 2); // Center Y coordinate
const radius = 150; // Radius of the circle
const speed = 5; // Speed of the mouse movement (adjust as needed)

const angle = (elapsedTime / 1000) * speed; // Angle increases over time
const x = centerX + Math.floor(radius * Math.cos(angle));
const y = centerY + Math.floor(radius * Math.sin(angle));

robot.moveMouse(x, y);

if (elapsedTime < 15000) { // Run for 30 seconds (30000 milliseconds)
elapsedTime += 50; // Increase elapsed time by 50 milliseconds for each movement
setTimeout(moveMouseInCircle, 50); // Call the function again after 50 milliseconds
}
}

// Call the function to start moving the mouse in a circle
moveMouseInCircle();
}
